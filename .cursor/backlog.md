# Бэклог фич (F1-F21)

## Реализованные

- **F14** «Согласование ретуши» в CRM-чате (2026-03-04)
- **F19** «Внутренний чат команды» (2026-03-05)

## В очереди

- **F1** Подтверждение телефона через SMS/Telegram-код
- **F2** 2FA через SMS для сотрудников
- **F3** ЛК партнёра: подтверждение расчётного счёта
- **F4** ЛК партнёра: спецссылки с автопромокодом (`?ref=CODE`)
- **F5** AI Chat: глубокие доработки Grok
- **F6** Voximplant: звонки из CRM
- **F7** Коннекторы: WhatsApp Business API прямой
- **F8** Типографии: API-подключение
- **F9** Принтеры из CRM (CUPS)
- **F10** Система бонусов клиентов
- **F11** AI-подсказчик для сотрудников
- **F12** Аудит лог сотрудников, защита клиентской базы
- **F13** Реферальная система «Порекомендуй другу»
  - Уникальная реферальная ссылка с кодом, скидка 10% для друга
  - Для «Фото на документы»: приложить фото до/после ретуши
  - Связано с F4, F10
- **F15** «Электронная сдача-приёмка кассы» (смена)
  - Слепая приёмка: принимающий НЕ видит сумму сдающего
  - Чаевые наличкой: отдельная строка, учёт при сверке
  - Расхождение > порога → уведомление руководителю
  - Связано с F12, POS
- **F16** «Редизайн layout CRM-дашборда»
  - Layout по частоте взаимодействия
  - Связано с F15, F11, Design System
- **F17** «Свой Telegram-бот» (замена Bitrix24)
  - grammy, webhook на `:3001/api/webhooks/telegram`
  - Inline-кнопки: Записаться | Статус заказа | Чат с оператором | Услуги
  - Канал `telegram` в CRM, пробрасывание сообщений + фото/файлы
  - Push-уведомления: заказ готов, напоминание, промо
  - Связано с F1, F7, F13
- **F18** «VK группа — ЛС в CRM» (замена Bitrix24 VK-коннектора)
  - Callback API (webhook), канал `vk` в CRM
  - `messages.send`, `markAsRead`, `markAsAnsweredConversation`
  - Фото/файлы через VK upload API
  - Связано с F7, F17, F12
- **F20** «Автоматическая регистрация reverse-SSH ключей устройств»
  - Windows/macOS/Linux установщики должны отправлять `TUNNEL_PUBKEY` на сервер автоматически
  - Убрать ручной шаг `SEND THIS TO CODEX` для пользователя
  - Сервер должен принимать ключ через защищённый endpoint/одноразовый enrollment token и сразу добавлять ограничение `permitlisten`
  - Связано с `public/downloads/svf-setup-windows-20260627.ps1`, `svf-enroll-tunnel-key`, reverse-SSH туннелями устройств
- **F21** «Современный bootstrap для настройки ПК с сервера»
  - Сделать маленький bootstrap/launcher: отправили сотруднику файл, он скачал, запустил, подтвердил UAC, дальше всё подтягивается с нашего сервера и настраивается автоматически
  - Поддержать server-driven выполнение актуального PowerShell-сценария от администратора: reverse-SSH, VPN, драйверы, ключи, device alias, обновления установщика
  - Убрать ручное копирование длинных one-liner команд и `SEND THIS TO CODEX`: launcher сам получает нужные параметры, отправляет публичный ключ и показывает понятный статус
  - Сделать это в современном формате 2026: подписанный launcher/MSIX/WinGet-пакет или легковесный агент, который умеет обновляться и запускать сценарии с сервера
  - Серверная сторона: манифест задач, версии сценариев, одноразовый enrollment token, лог выполнения, возможность выдать конкретный сценарий для конкретного ПК/сотрудника
  - Сохранить практичный “хакерский” UX: минимум вопросов, один запуск, дальше автоматическая настройка без ручных инструкций в чате
  - Связано с F20 и установщиками `public/downloads/svf-setup-*.ps1`
